Home Lab Automation Platform

A self-hosted server that connects workflow automation, secure remote access, and an AI assistant — built to eventually read and draft business documents for a small electrical company.

n8n Cloudflare Tunnel Docker Microsoft Graph API MCP
workstation → Cloudflare Tunnel → n8n / MCP bridge → Microsoft Graph (OneDrive & Outlook)

The problem

FoxElectric, a small electrical business, keeps its templates and records in OneDrive. The goal was to let an AI assistant read those documents and draft new ones — quotes, records, correspondence — without manually copying files back and forth, and without exposing anything to the open internet.

What's built so far

  • n8n deployed in Docker on a repurposed Dell OptiPlex 7060 running Ubuntu/Debian, as the workflow automation engine.
  • A domain (electricalfox.com) registered through Porkbun, with DNS managed in Cloudflare.
  • A Cloudflare Tunnel routing subdomains to the home server, so nothing needs an inbound port open on the home network.
  • An Azure App Registration with Microsoft Graph delegated permissions, allowing secure OAuth2 access to OneDrive and Outlook.
  • A working "Search Outlook Emails" workflow, tested end-to-end through Claude Desktop.

A real obstacle

n8n's built-in MCP feature ran into OAuth redirect URI issues that blocked a clean connection to Claude Desktop. Rather than fighting the built-in implementation, a separate n8n-mcp bridge container was stood up to sit between n8n and the MCP client — a practical workaround that kept the project moving instead of stalling on one tool's limitation.
